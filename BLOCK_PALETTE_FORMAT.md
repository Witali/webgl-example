# BPAL v2

`BPAL` — компактный формат блочно-палитрового изображения. Все многобитовые
значения записываются от старшего бита к младшему. Декодер также поддерживает
файлы первой версии.

## Заголовок v2

Первые 4 байта — ASCII magic `BPAL`. За ним следует 80-битное поле заголовка:

| Смещение, бит | Размер | Значение |
| ---: | ---: | --- |
| 0 | 4 | Версия формата, для v2 равна `2` |
| 4 | 24 | Ширина минус 1 |
| 28 | 24 | Высота минус 1 |
| 52 | 3 | `log2(размер блока) - 1` |
| 55 | 2 | `log2(цветов в блоке) - 1` |
| 57 | 4 | `log2(цветов общей палитры) - 1` |
| 61 | 1 | Формат цвета: `0` — RGB565, `1` — RGB888 |
| 62 | 1 | Модель палитры: `0` — явная, `1` — RGB-векторы |
| 63 | 9 | Число векторов минус 1; для явной палитры равно нулю |
| 72 | 8 | Зарезервировано, в v2 равно нулю |

Итого служебный заголовок v2 занимает 14 байт: 4 байта magic и 10 байт битовых
полей.

## Полезная нагрузка

После заголовка без промежуточного выравнивания идут:

1. Описание общей палитры:
   - для явной модели — все цвета по 16 или 24 бита;
   - для векторной модели — начальный и конечный цвет каждого RGB-вектора.
2. Палитры блоков в построчном порядке блоков: на каждый индекс выделяется
   `log2(число цветов общей палитры)` бит.
3. Локальные индексы всех пикселей в построчном порядке изображения: на индекс
   выделяется `log2(число цветов в блоке)` бит.

Цвета общей векторной палитры восстанавливаются линейной интерполяцией. На
каждый вектор сначала выделяется `floor(число цветов / число векторов)` цветов,
остаток по одному распределяется между первыми векторами. Обе конечные точки
включаются в полученную последовательность.

Только последний байт файла при необходимости дополняется нулевыми битами.
Альфа-канал не хранится; декодированное изображение считается полностью
непрозрачным.

## Совместимость с v1

Заголовок v1 после magic занимает 64 бита и заканчивается двумя нулевыми
резервными битами. В первой версии палитра всегда записана явно. Расположение
индексов блоков и пикселей совпадает с v2.
